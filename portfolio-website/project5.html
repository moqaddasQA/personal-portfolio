Project 5: Jenkins CI/CD Integration

This project demonstrates how to integrate automated tests with Jenkins for Continuous Integration and Continuous Delivery (CI/CD). It showcases the creation of a Jenkins pipeline that builds the project, runs the tests, and publishes the test reports. The project includes a sample Jenkinsfile and instructions for setting up a GitHub webhook to trigger builds automatically.

Key Features

  • Jenkins Pipeline: Defines a CI/CD pipeline using a Jenkinsfile.
  • Automated Builds: Automates the build process using Maven.
  • Automated Testing: Integrates automated tests written with TestNG.
  • Test Reporting: Publishes TestNG reports to Jenkins for easy access.
  • GitHub Integration: Triggers builds automatically on code push events via a GitHub webhook.

Technologies Used

  • Jenkins
  • Maven
  • Git
  • GitHub
  • TestNG

Workflow

The workflow for this project includes:

  • Code Commit: A developer commits code to a GitHub repository.
  • Trigger Build: A GitHub webhook notifies Jenkins of the commit.
  • Checkout Code: Jenkins checks out the code from the repository.
  • Build and Test: Jenkins builds the project and runs the automated tests using Maven.
  • Report Results: Jenkins report the test results.

Jenkinsfile

The Jenkins file is used to implement the different stages of the process

Conclusion

This project is a great example of how we can use Jenkins to have a successful CI/CD process
