Project 2: API Testing Suite

This project is designed to demonstrate API testing skills using a variety of tools and technologies. It focuses on creating a robust suite of automated tests for a RESTful API, ensuring the reliability and functionality of its endpoints. The project emphasizes best practices such as data-driven testing and continuous integration.

Key Features

  • Comprehensive API Testing: Automates tests for a variety of API endpoints, covering essential HTTP methods (GET, POST, PUT, DELETE).
  • Data-Driven Testing: Uses external data sources to feed test data, allowing for a wide range of scenarios to be tested.
  • Reliability Focus: Includes test cases that ensure API reliability, such as checking response times and validating status codes.
  • CI/CD Integration: Contains a Jenkinsfile for seamless integration with Jenkins pipelines.
  • Detailed Reporting: Generates detailed test reports to help understand test results.
  • Error Handling: Implements robust error handling to identify issues in both the API and the test suite.

Tools & Technologies Used

  • REST Assured: For writing and executing API tests.
  • TestNG: For test execution and reporting.
  • Jackson: For handling JSON data.
  • Maven: For build and dependency management.
  • Jenkins: For Continuous Integration/Continuous Delivery (CI/CD).
  • SLF4j: For logging.

API Endpoints

This project tests the following mock API endpoints:

  • GET /api/products: Retrieves a list of all products.
  • GET /api/products/{id}: Retrieves a single product by ID.
  • POST /api/products: Creates a new product.
  • PUT /api/products/{id}: Updates an existing product.
  • DELETE /api/products/{id}: Deletes a product.

Test Cases

The test suite covers a wide range of scenarios, including:

  • GET requests: Verifies the ability to retrieve products.
  • POST requests: Checks product creation.
  • PUT requests: Ensures products can be updated.
  • DELETE requests: Confirms product deletion.
  • Error Handling: Validates appropriate error responses for invalid requests.
  • Performance: Measures and validates response times.

Data-Driven Testing

The TestData class provides various data to the ProductApiTest class, including sample products and product IDs. Using TestNG's data provider to provide multiple sets of data for creating, updating, and deleting products.

Jenkins Integration

The project includes a Jenkinsfile for easy integration with Jenkins CI/CD. The Jenkinsfile includes steps to clean the project and run the tests.
