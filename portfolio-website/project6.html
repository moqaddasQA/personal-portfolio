Overview

This project demonstrates best practices for version control using Git and GitHub, along with a simulation of Agile tools and processes, inspired by Jira.

Key Features

  • Git Workflows: Demonstrates branching, merging, and pull requests.
  • GitHub Collaboration: Shows how to collaborate using GitHub.
  • Simulated Jira: Includes a simple text file to simulate Jira issue tracking.
  • Reporting: Provides a script to generate a report from simulated Jira issues.
  • Git log: Provides a script to get the history of the git log.

Tools and Technologies

  • Git: For version control.
  • GitHub: For hosting the repository and collaboration.
  • Jira (Simulated): Using a text file to represent Jira issues.
  • Shell Scripting: For generating reports and git log.

Git Usage

The project demonstrates key Git commands such as git init, git clone, git add, git commit, git push, git pull, git branch, and git merge. The workflow includes:

  • Creating new branches for feature development.
  • Committing changes to feature branches.
  • Pushing changes to a remote repository.
  • Creating and merging pull requests.

Simulated Jira

The jira_issues.txt file is used to simulate Jira issue tracking. Each line in the file represents an issue with the following format: IssueID | Summary | Description | Status.

The jira_report.sh script reads this file and generates a summary report on open, in progress, and closed issues.

Git log
